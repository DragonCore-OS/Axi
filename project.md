## T0 - 时间盒规则 (Time-Box Rule)

### 规则定义

**所有实验不得超过现实时间 1 小时。**

### 自动批准与续时

Ralph **可以自动批准并继续**，但阈值必须严格到**人工确认级别**。

### 自动批准标准（严格阈值）

| 条件 | 阈值 | 动作 |
|------|------|------|
| **POSITIVE_AUTO** | transfer_gap ≥ **10pp** AND retention ≥ **90%** | **自动批准，继续下一小时** |
| **POSITIVE_MANUAL** | transfer_gap ≥ 5pp AND retention ≥ 85% | 生成申请，等待人工确认 |
| **MARGINAL** | 0 < transfer_gap < 5pp | 冻结，等待分析 |
| **FAIL** | transfer_gap ≤ 0 OR retention < 85% | 冻结，终止或回退 |

### 自动批准流程

```
实验运行 1 小时
    ↓
[Ralph] 强制 STOP (timeout 3600)
    ↓
[评估阈值]
    ↓
├── POSITIVE_AUTO (≥10pp, ≥90%) 
│   → 自动批准
│   → 生成 hour2_config.json
│   → **自动继续 Hour-2**
│   → 记录决策日志
│
├── POSITIVE_MANUAL (≥5pp, ≥85%)
│   → 生成申请材料
│   → 等待人工确认
│
└── MARGINAL / FAIL
    → 冻结
    → 等待人工决策
```

### 关键纪律

- ✅ **严格阈值**: 自动批准标准必须远高于最低门槛
- ✅ **可审计**: 每次自动批准必须记录完整决策链
- ✅ **可回滚**: 即使自动批准，后续发现异常仍可冻结
- ❌ **放宽标准**: 不得降低自动批准阈值

### 正向反馈标准

| 类型 | 示例 |
|------|------|
| 技术验证 | 核心算法通过测试 |
| 数据确认 | 实验假设得到数据支持 |
| 问题定位 | 阻塞问题已识别并有解决方案 |
| 性能突破 | 关键指标达到预期 |

### 强制执行

- ⏰ **1 小时硬截止**：无论状态如何，必须停止并汇报
- 📝 **强制记录**：终止时必须记录当前状态、已得成果、未完成项
- 🚫 **无例外**：未经申请的延长视为违规
